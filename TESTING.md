# ЁЯзк Testing

## Philosophy

Pure logic tests cover everything that can break without a real node.
Integration ("testei na carne") covers I/O, performance, and full-chain sync.

**Neither replaces the other.** Each has its place.

---

## Quick Start

```bash
# Run everything
cargo test

# Run only pure unit tests
cargo test --lib

# Run integration tests (chain_reader fixture)
cargo test --test chain_reader_integration

# List all tests by name
cargo test -- --list
```

All tests run in **< 1 second** and never touch `~/9c-blockchain`.

## What's NOT Tested (intentional)

The following modules need real I/O or a production dataset. They are validated
by syncing a real node on a test hard drive ("testei na carne"):

| Module | Why excluded |
|---|---|
| `exporter.rs` тАФ `export_states()` | Requires a `states/` DB with millions of entries |
| `pruner.rs` тАФ `prune_states()` | Requires export.bin + live_keys + dest DB |
| `gc_filter.rs` тАФ `bfs_fixpoint()` | Requires a multi-GB export file |
| `checkpoint_secondary.rs` | Requires a RocksDB with WAL / memtable |
| `snapshot.rs` тАФ `create_snapshot()` / `create_single_archive()` | Tar + Zstd multi-thread I/O |
| `verify.rs` тАФ `verify_archive()` | Requires a real `.tar.zst` archive |
| `node_detect.rs` тАФ `check_node_running()` | Requires `flock()` on real LOCK files |

## Fixture Regeneration

If `tests/fixtures/mini-chain/` is ever deleted or corrupted:

```bash
cargo run --example generate_fixture
```

This creates a synthetic 3-block RocksDB chain inside the repo.
Zero external dependencies.
